### Ответ:
Основная проблема - *утечка памяти из-за среза* 

Что происходит с justString?
В someFunc(), v — это очень длинная строка ( 1024 символа).
При выполнении justString = v[:100], Go создает новую строку justString, которая имеет длину 100 символов. Но, для повышения эффективности, justString продолжает ссылаться на тот же базовый массив байтов, что и исходная строка v.
После выхода из someFunc(), сборщик мусора (GC) не может освободить память, занятую строкой v целиком, потому что глобальная переменная justString все еще ссылается на ее начало. В результате, 924 байта (1024 - 100) остаются занятыми, хотя и не используются.


### Как исправить ?

Самый надежный способ — преобразовать срез строки в массив байтов, а затем обратно в строку. Это заставляет Go выделить новый, независимый участок памяти.

```go
justString = string([]byte(v[:100]))
```
